# Complete Railway.app Setup Guide

Deploy Student Advising Portal to Railway with persistent PostgreSQL database.

---

## **🚀 Step 1: Deploy Application to Railway**

### **A. Create New Project**

1. Go to: https://railway.app
2. Sign up with GitHub (or login)
3. Click **"New Project"**
4. Click **"Deploy from GitHub repo"**
5. Select: `tanvir-hannan-anik/Student-Advising`
6. Click **"Deploy"**

Railway will build and deploy automatically using the Dockerfile!

### **B. Get Your Public URL**

After deployment (3-5 minutes), you'll see:
```
✓ Your app is live at: https://student-advising-xxxxx.up.railway.app
```

---

## **🗄️ Step 2: Add PostgreSQL Database**

### **A. Create Database Service**

1. In your Railway project, click **"+ New Service"**
2. Select **"Database"** → **"PostgreSQL"**
3. Railway auto-creates and initializes the database
4. You'll get `DATABASE_URL` environment variable automatically

### **B. Verify Database Connection**

In **Variables** tab, you should see:
```
DATABASE_URL = postgresql://user:password@host:port/railway
```

---

## **Step 3: Configure Environment Variables**

Go to **Variables** tab and add:

### **Required:**
```
GROQ_API_KEY = [get from https://groq.com]
```

### **Optional (for email features):**
```
GMAIL_USER = your_email@gmail.com
GMAIL_APP_PASSWORD = [16-char app password from Google]
```

**Note:** `DATABASE_URL` is auto-generated by Railway ✓

---

## **Step 4: Redeploy Application**

After adding database:

1. Click **"Redeploy"** button
2. Wait 3-5 minutes for build
3. Watch logs to verify:
   ```
   ✓ Connected to PostgreSQL
   ✓ Tables created
   ✓ Application started on port 8080
   ```

## **🐛 Troubleshooting**

### **App Won't Start**
```
Error: Failed to connect to database
```
**Solution:**
- Ensure PostgreSQL service is running
- Check `DATABASE_URL` is set in Variables
- Redeploy after adding database

### **Data Not Persisting**
**Solution:**
- Verify PostgreSQL service exists
- Check application logs for connection errors
- Wait 5 minutes after creating database

### **Slow First Load**
- First load downloads 300MB dependencies
- Normal and takes 2-5 minutes
- Subsequent requests are fast

### **420 Too Many Requests (AI Service)**
- Groq API rate limit reached
- Wait a few minutes or upgrade Groq account
